# あいうえバトル Webアプリ 仕様書（確定版）

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | あいうえバトル Online |
| 目的 | 身内同士でオンライン対戦を楽しむ |
| ホスティング | GitHub Pages（静的サイト） |
| バックエンド | Firebase（Authentication / Realtime Database） |
| プレイ人数 | 2〜6人（可変） |
| 対応デバイス | スマホ・PC（レスポンシブ） |

---

## 2. 技術構成

### 2.1 フロントエンド
- **React**（状態管理の複雑さに対応）
- GitHub Pages でホスティング

### 2.2 Firebase サービス

| サービス | 用途 |
|----------|------|
| Authentication | 匿名認証（簡易ログイン、ニックネーム設定） |
| Realtime Database | ルーム管理・ゲーム状態の同期 |

### 2.3 使用しないサービス
- Firestore / Storage / Functions は不使用（シンプル構成を維持）

## 4. 画面別 詳細仕様

### 4.1 トップ画面

**表示要素：**
- アプリタイトル・ロゴ
- ニックネーム入力欄
- 「ルーム作成」ボタン
- 「ルーム参加」ボタン（ルームID入力フィールド付き）
- 「ルール説明」ボタン

**処理：**
- ニックネーム未入力時はルーム作成・参加を無効化
- Firebase匿名認証を実行し、UIDを取得
- ニックネームはlocalStorageにも保存し次回自動入力

---

### 4.2 ルール説明画面

**表示要素：**
- ルール仕様書の内容を分かりやすく表示（イラスト・アイコン付き推奨）
- 「戻る」ボタン

**セクション構成：**
1. ゲームの目的（1文で）
2. 準備：お題を決めて単語を入力
3. 表記変換ルール（濁点除去・小文字変換等）＋具体例
4. バトルの流れ（攻撃→判定→連続or交代）
5. 脱落・勝利条件
6. Tips（当てにくい単語の選び方）

---

### 4.3 ルーム作成 → ロビー画面

**ルーム作成処理：**
- 4桁のルームID（英数字）を自動生成
- Realtime Database に `/rooms/{roomId}` を作成
- 作成者を `hostUid` として記録

**ロビー画面の表示要素：**
- ルームID（共有用、コピーボタン付き）
- 参加者一覧（リアルタイム更新）
- 参加人数表示（現在 X / 6 人）
- 「ゲーム開始」ボタン（ホストのみ表示、2人以上で有効化）
- 「ルームを閉じる」ボタン（ホストのみ）

**バリデーション：**
- 最大6人まで。満員時は参加を拒否
- ホスト退出時はルーム解散（全員をトップに戻す）

---

### 4.4 ルーム参加 → ロビー画面

**参加処理：**
- ルームIDを入力して参加
- 存在しないルーム / 満員 / ゲーム中 の場合はエラー表示
- 参加成功後はロビー画面へ遷移

**ロビー画面の表示要素（ゲスト）：**
- ルームID
- 参加者一覧
- 「ホストがゲームを開始するのを待っています...」メッセージ
- 「退出」ボタン

---

### 4.5 お題設定画面

**お題設定者の決定ロジック：**

| 条件 | お題設定者 |
|------|-----------|
| 初回（第1ゲーム） | ランダムで1人を選出 |
| 第2ゲーム以降 | 前ゲームの**最下位**プレイヤー（最も早く脱落した人） |
| 最下位が同率脱落 | その中からランダムで1人 |

**表示要素：**
- 「○○さんがお題を設定します」表示
- お題設定者 → 自由テキスト入力欄＋「決定」ボタン
- お題設定者以外 → 「○○さんがお題を考えています...」待機表示
- お題候補リスト表示（タップで入力補完）

**お題候補リスト（初期実装分）：**
- 動物 / 飲みもの / 食べもの / ポケモン名 / 果物 / 国名 / スポーツ / コンビニにあるもの / 赤いもの / 行きたい場所 / 乗りもの / キャラクター名 / 文房具 / 4文字の言葉

---

### 4.6 単語入力画面

**表示要素：**
- 現在のお題（上部固定表示）
- 8枠の入力スロット（左から1〜8）
- ひらがなキーボード（50音表形式、清音＋「ん」＋長音符号「ー」）
- 「✕」ボタン（空きスロットを埋める用）
- 「全消し」ボタン
- 「決定」ボタン
- 他プレイヤーの入力状況（「入力済み ✓ / 入力中...」リアルタイム表示）

**入力方式：**
- 端末IMEは使用せず、**アプリ内50音キーボード**で入力
- キーボードには清音＋「ん」＋「ー」のみ配置
- 濁点・半濁点・小文字キーが存在しないため、変換ミスが構造的に発生しない

**バリデーション：**
- 最低2文字（✕以外）が必要
- 全8スロットが埋まるまで「決定」ボタンは無効（✕で埋めてもOK）
- 決定後の変更は不可
- タイムアウトなし（全員が入力するまで待つ）

**遷移：**
- 全員の `inputReady` が `true` になった時点でバトル画面へ自動遷移

---

### 4.7 バトル画面（メイン）

**画面レイアウト（参考UIベース）：**

```
┌─────────────────────────────────────┐
│            お題：ポケモン名           │
├──────┬───────────────────┬──────────┤
│ P1   │                   │ P2       │
│[手札]│   50音ボード       │ [手札]   │
├──────┤  （攻撃エリア）     ├──────────┤
│ P3   │                   │ P4       │
│[手札]│                   │ [手札]   │
├──────┴───────────────────┴──────────┤
│         手番：○○さん                 │
└─────────────────────────────────────┘
```

**50音ボード：**
- 未使用文字 → タップ可能（手番プレイヤーのみ）
- 使用済み文字 → ハイライト＋タップ無効
- 「を」「ん」「ー」を含む

**手札エリア（各プレイヤー）：**
- 8枠表示
- 非公開スロット → 番号のみ表示
- 公開済みスロット → 文字を表示
- 脱落プレイヤー → グレーアウト
- 退出済みプレイヤー → グレーアウト（手札は非公開のまま）

**手番順：**
- 初回はランダムで決定（ゲーム開始時にシャッフル）
- 2ゲーム目以降も毎回ランダムで再シャッフル
- 脱落・退出済みプレイヤーはスキップ

**手番の処理フロー：**

```
手番プレイヤーが50音ボードの文字をタップ
  │
  ├─ 全プレイヤーの手札を自動判定
  │
  ├─ ゲーム終了チェック（※毎攻撃後に最優先で実行）
  │    └─ 残り1人になった → 即ゲーム終了（連続攻撃が残っていても実行しない）
  │
  ├─ 勝利確定チェック
  │    └─ 自分以外が全員脱落済み → 自分の勝利確定（以降の自爆では脱落しない）
  │
  ├─ ヒット判定
  │    ├─ 他プレイヤーにヒット有り
  │    │    ├─ 該当スロットを全てオープン（同じ文字が複数あれば全て）
  │    │    ├─ 脱落判定（✕以外かつrevealed=falseのスロットが0か？）
  │    │    ├─ 攻撃者自身が脱落した場合 → 手番終了（連続攻撃なし）
  │    │    └─ 攻撃者が生存中：
  │    │         1回目の攻撃 → 連続攻撃（もう1回）
  │    │         2回目の攻撃 → 脱落判定を実行した上で手番終了
  │    │
  │    ├─ 自爆のみ（自分だけヒット、他プレイヤー無傷）
  │    │    ├─ 自分の該当スロットをオープン
  │    │    └─ 手番終了（連続攻撃なし）
  │    │
  │    ├─ 他プレイヤーヒット ＋ 自爆
  │    │    ├─ 全員の該当スロットをオープン
  │    │    ├─ 攻撃者自身が脱落した場合 → 手番終了（連続攻撃なし）
  │    │    └─ 攻撃者が生存中 → 連続攻撃あり（他プレイヤーへのヒットとして扱う）
  │    │
  │    └─ 誰にもヒットなし → 手番終了
  │
  └─ 次の手番プレイヤーへ（脱落者・退出者はスキップ）
```

**ゲーム終了判定：**
- 残り1人になった時点で即終了（連続攻撃の途中でも打ち切り）
- 同時脱落時は攻撃を行ったプレイヤーが勝利（引き分けなし）
- 最後の1人は勝利確定状態となり、自爆しても脱落しない

**タイムアウト：**
- なし（身内利用のため無制限）

---

### 4.8 結果画面

**表示要素：**
- 順位一覧（1位 = 最後まで残った人、以下脱落順の逆）
- 同時脱落は同率順位で表示
- 各プレイヤーの単語を全公開（変換前の元の単語ではなく、入力された清音表記を表示）
- 「次のゲームへ」ボタン（**ホストのみ表示**）→ お題設定画面へ遷移
- 「ルームを出る」ボタン（全員表示）

**順位記録：**
- 脱落順を `eliminatedOrder` で記録
- 最下位（最も早く脱落 = `eliminatedOrder` が最小）を次ゲームのお題設定者に引き継ぐ
- 途中退出者は順位対象外（お題設定者にもならない）
- お題設定者に選ばれたプレイヤーが「次のゲームへ」前に退出した場合 → 残りメンバーからランダムで再選出

## 5. データ設計（Realtime Database）

### 5.1 ルーム構造

```
/rooms/{roomId}/
  ├── status: "lobby" | "setting_topic" | "inputting" | "battle" | "result"
  ├── hostUid: "uid_xxx"
  ├── topic: "ポケモン名"
  ├── topicSetterUid: "uid_yyy"
  ├── gameCount: 1
  ├── turnIndex: 0                    // playerOrder 配列内の現在のインデックス
  ├── turnAttackCount: 0              // 現在の手番での攻撃回数（0 or 1）
  ├── playerOrder: ["uid_a", "uid_b", "uid_c", ...]
  ├── lastActivity: 1711234567890     // 最終活動タイムスタンプ（ルーム寿命管理用）
  │
  ├── players/
  │     ├── {uid}/
  │     │     ├── name: "こうき"
  │     │     ├── slots: ["ひ","か","ち","ゆ","う","✕","✕","✕"]
  │     │     ├── revealed: [false, false, true, false, false, false, false, false]
  │     │     │   // ※ ✕スロットのrevealedは常にfalse。脱落判定では✕スロットを除外して判定する
  │     │     ├── isEliminated: false
  │     │     ├── eliminatedOrder: null     // 脱落順（1=最初に脱落、同時脱落は同番号）
  │     │     ├── inputReady: false
  │     │     └── isDisconnected: false     // 切断フラグ
  │     └── ...
  │
  ├── board/
  │     ├── あ: false      // false=未使用, true=使用済み
  │     ├── い: false
  │     ├── ...
  │     ├── を: false
  │     ├── ん: false
  │     └── ー: false
  │
  └── log/                  // 攻撃ログ（任意、リプレイ用）
        ├── 0: { uid: "uid_a", char: "い", hits: ["uid_b"], timestamp: ... }
        └── ...
```

### 5.2 攻撃判定のアーキテクチャ

**方針：`slots` を全プレイヤーに読み取り可とし、クライアント側で判定する。**

- 身内利用のため、DevTools等でのカンニングリスクは許容する
- UIレベルでは非公開スロットの文字を表示しない
- `slots` のFirebaseリスナーは自分のデータのみ常時リッスンし、他プレイヤーの `slots` は攻撃判定時と結果画面遷移時にのみ参照する（不意に見えることを防止）
- `slots` の書き込みは本人のみに制限する（セキュリティルールで制御）

### 5.3 セキュリティルール（MVP）

```json
{
  "rules": {
    "rooms": {
      "$roomId": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    }
  }
}
```

**MVP方針：**
- ルーム全体を `auth != null` で読み書き可とし、シンプルに保つ
- `slots` の本人限定書き込み等の細分化はPhase 2以降で対応
- 身内利用のため、意図的な不正書き込みリスクは許容する

### 5.4 「次のゲームへ」時のリセット処理

ホストが「次のゲームへ」を押下した際に、以下をアトミックに更新する：

```
{
  "status": "setting_topic",
  "topic": null,
  "topicSetterUid": "{最下位プレイヤーのUID}",
  "gameCount": {前回+1},
  "turnIndex": 0,
  "turnAttackCount": 0,
  "playerOrder": ["{ランダム再シャッフル}"],
  "lastActivity": {現在時刻},
  "board/あ": false,
  "board/い": false,
  ... （全文字を false に）
  "players/{uid_a}/slots": null,
  "players/{uid_a}/revealed": [false,false,false,false,false,false,false,false],
  "players/{uid_a}/isEliminated": false,
  "players/{uid_a}/eliminatedOrder": null,
  "players/{uid_a}/inputReady": false,
  ... （全プレイヤー分）
}
```

## 6. ゲームフロー状態遷移

```
lobby （ロビー）
  │  ホストが「ゲーム開始」
  ▼
setting_topic （お題設定）
  │  設定者がお題を入力して「決定」
  ▼
inputting （単語入力）
  │  全員が入力完了（タイムアウトなし）
  ▼
battle （バトル中）
  │  残り1人になる
  ▼
result （結果表示）
  │  ホストが「次のゲームへ」
  ▼
setting_topic （次ゲームのお題設定）
  ...ループ
```

**状態遷移のトリガー：**

| 遷移 | トリガー | 実行者 |
|------|----------|--------|
| lobby → setting_topic | 「ゲーム開始」ボタン | ホスト |
| setting_topic → inputting | お題「決定」ボタン | お題設定者 |
| inputting → battle | 全員の `inputReady` が `true` | 自動 |
| battle → result | 残り1人（ゲーム終了判定） | 自動 |
| result → setting_topic | 「次のゲームへ」ボタン | ホスト |

## 7. 非機能要件

### 7.1 リアルタイム同期
- Firebase Realtime Database の `onValue` / `onChildChanged` でリアルタイムリスナー
- 手番プレイヤーの攻撃 → 即座に全クライアントに反映
- 他プレイヤーの `slots` は常時リッスンしない（攻撃判定時のみ参照）

### 7.2 切断対応
- `onDisconnect` で `players/{uid}/isDisconnected` を `true` に設定
- バトル中の切断 → 該当プレイヤーは自動脱落扱い（手札は非公開のままグレーアウト）
- ホスト切断 → ルーム解散（全員をトップに戻す）

### 7.3 途中参加/途中退出
- バトル中（`status` が `battle` / `inputting` / `setting_topic`）の途中参加は不可
- ロビー（`status` が `lobby`）でのみ参加可能
- 退出したプレイヤーは順位対象外、お題設定者にもならない

### 7.4 ルームの寿命
- `lastActivity` タイムスタンプを状態遷移ごとに更新
- MVP段階では手動削除は不要（Firebase無料枠で容量的に問題なし）
- 将来的に定期クリーンスクリプトで `lastActivity` が1時間以上前のルームを削除

### 7.5 レスポンシブ対応
- スマホ縦画面をメイン想定
- PC横画面でも崩れないレイアウト
- 50音ボードはスマホでもタップしやすいサイズ確保

| は行 | は | ひ | ふ | へ | ほ |
| ま行 | ま | み | む | め | も |
| や行 | や | | ゆ | | よ |
| ら行 | ら | り | る | れ | ろ |
| わ行 | わ | | を | | ん |
| 記号 | ー | | | | |

### 8.2 単語入力用キーボード構成
- 上記ボードと同一の文字セット（清音＋「ん」＋「ー」）
- 追加ボタン：「✕」（空きスロット用）、「←」（1文字削除）、「全消し」

## 9. 実装優先度

### Phase 1（MVP）
- トップ画面（ニックネーム入力・ルーム作成/参加）
- ロビー画面（参加者一覧・ゲーム開始）
- お題設定画面（初回ランダム選出・最下位ロジック）
- 単語入力画面（アプリ内50音キーボード）
- バトル画面（攻撃・判定・自爆・連続攻撃・脱落・勝利）
- 結果画面（順位・全単語公開・次のゲームへ）
- ルール説明画面

### Phase 2（改善）
- 攻撃アニメーション・SE
- 切断対応（onDisconnect）
- レスポンシブ最適化
- ルームの自動削除

### Phase 3（追加機能）
- 戦績記録（連勝数、プレイ回数）
- 英語モード（アルファベットボード）
- 観戦モード
- カスタムお題リスト管理
