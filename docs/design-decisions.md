## 追加レビュー決定事項

| ID | 論点 | 決定内容 | 理由 |
|----|------|----------|------|
| C4 | 自爆で自分が脱落した場合の連続攻撃 | 行えない（脱落時点で手番終了） | 脱落者がアクションを取れるのは不自然 |
| C5 | 最後の1人が自爆した場合 | 勝利確定状態のため脱落しない | 残り0人は許容しない |
| C6 | 残り1人になった瞬間のゲーム終了 | 即終了（連続攻撃途中でも打ち切り） | 無駄な攻撃を防ぐ |
| C7 | お題設定者が退出した場合 | 残りメンバーからランダム再選出 | ゲーム続行のため |
| C8 | セキュリティルール（MVP） | ルーム全体に auth!=null（細分化は後回し） | 開発速度優先 |

## 技術的判断

| 判断 | 内容 | 理由 |
|------|------|------|
| CSS設計 | 単一App.cssに集約 | CSSモジュール不要な規模。コンポーネント数が少ない |
| React Router不使用 | useStateベースの画面遷移 | Firebaseの`status`フィールドで画面が決まるため不要 |
| 50音キーボード | アプリ内カスタムキーボード | 濁点・半濁点の変換ミスを構造的に防止 |
| Firebase操作の集約 | roomService.jsに全集約 | コンポーネントからFirebase直接importを禁止 |
